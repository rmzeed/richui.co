The Gen-UI Playbook.

A new kind of interface is here. One that doesn't exist until a person asks for it. This is a calm, considered place to start designing for it — the foundations, the building blocks, and the decisions that only a designer can make.

01Foundations

Thinking in systems.

Generative UI doesn't change what design is. It changes the unit of it. Where you used to design every screen, you now design the vocabulary that the screens are composed from. That shift is the quietest and the most important.

Shift 01

Screens → systems

You can't draw every screen. There are infinite combinations. You define the primitives and the rules; the model assembles.

Shift 02

Path → intent

Forget “tap here, then here, then here.” Design for user intent. The model picks the path.

Shift 03

Fixed → ephemeral

Every UI is a one-time composition. Stop optimising for reuse; optimise for the single moment it appears.

A useful shorthand

The answer a person sees is a function of three things: the primitives you've made available, the context they're in, and the intent behind their question. Most of the work is in the first.

03Primitives

Five families of primitives.

A generative UI is composed from a library of pre-built pieces — primitives. The strong ones come in five families. Visual primitives create perception. Semantic primitives create meaning. Content primitives shape the answer. Interaction primitives let people steer. Trust primitives help them believe.

Family 01

Visual primitives

Create perception

Color, type size, spacing, elevation, border, icon, motion, surface, contrast. The pure perceptual layer. They guide attention before anyone reads anything.

ColorTypeSpacingElevationMotion
Family 02

Semantic primitives

Create meaning

Status, priority, warning, recommendation, evidence, comparison, progress, uncertainty, source, action. Deeper than visual — these carry a role.

StatusPriorityWarningEvidenceUncertainty
Family 03

Content primitives

Shape the answer

Summary, reason, metric, date, amount, ranking, caveat, next step, source, exception. The named parts of an answer — each gets its own visual role.

SummaryMetricRankingCaveatSource
Family 04

Interaction primitives

Let the user steer

Expand, compare, filter, edit, confirm, undo, save, dismiss, give feedback, ask follow-up. Stop the one-way broadcast. Turn it into a dialogue.

ExpandFilterEditUndoAsk
Family 05

Trust primitives

Calibrate belief

Confidence label, “last updated” note, source, evidence, assumption, missing data, correction path. The primitives that earn the user's belief.

Every component picks from all five

A well-formed gen-UI component is never just a visual card. It is also a status, an evidence trail, a possible action, and a correction path — all at once.

05Walkthrough

Building one, end to end.

Let's design a real component — a Sales Insight widget that appears inside a CRM chat when a salesperson asks about a deal. Nine steps. Each one a real decision. Tap any to expand.

Step 01

Define the surface

Where does this component live?

Before drawing anything, decide the surface. A gen-UI component for a chat sidebar behaves very differently than one in a full-page canvas.

For Sales Insight we pick: inside a CRM chat thread, right-rail. Always less than 440 px wide. Scrollable. Sits next to other agent messages.

The surface defines: the maximum width, the typographic scale, the entrance/exit choreography, the expected reading time, and what the component sits next to.

440px maxSidebar~10s readStacks with messages
Step 02

Pick the intent

One of the seven — say it as a sentence.

Choose from the seven: inform · compare · decide · act · monitor · recover · explain. Then write the user's sentence.

For Sales Insight: inform + decide. The sentence is “Tell me what's going on with the Acme deal and what I should do next.”

This sentence becomes the single source of truth. Every primitive, every state — does it serve this sentence?

Step 03

Pick from the pantry

Which primitives from which families?

Now go shopping in your library. For Sales Insight you pick across all five families: from content — title, metric, ranking; from semantic — risk priority, recommendation; from visual — color tokens for risk levels; from interaction — expand, refine; from trust — source, freshness, confidence.

Six pieces total. Six is great. Sixty is a trap.

TitleMetric ×3Risk calloutActionSource
Step 04

Define the slots

Where can the model fill in data?

A slot is a hole. The model puts something in it. The schema says what shape that something must be.

For Sales Insight: title (deal name, max 40 chars), metrics (1 to 4 tiles, each with name + value + delta), risk (one of: low / medium / high, with a sentence), action (label + payload). Source link is always required.

Designing slots is the heart of the work. Too rigid and the answer is boring; too loose and the model hallucinates layout.

Step 05

Set the guardrails

Tone, density, allowlists, tokens.

Three kinds of guardrail: visual (brand tokens only, locked type scale), tone (if data shows critical risk, forbid “playful” primitives), density — simple answers get a Micro-UI, complex answers expand to a Macro-UI.

Density logic is the under-rated lever: the same component should be able to compress or expand depending on how much the answer actually carries.

Step 06

Design every state

Idle, loading, streaming, complete, error, refining.

This is where most gen-UI components fail. You design the happy path and ship the rest as raw JSON or a spinner.

Every gen-UI component needs at minimum: idle · loading · streaming · complete · error · refining. For each, design what's visible, what's interactable, and what's moving.

Step 07

Choreograph the motion

The sequence that earns user attention.

Streaming UI is a performance, not a screen. Choreography is the sequence in which the parts arrive.

For Sales Insight: at 0ms the frame fades in. At 120ms the title types in. At 240ms the first metric tile lifts. Each subsequent metric staggers by 80ms. The risk callout fades at 520ms. The action button slides up last, at 720ms. The source line appears in the same beat.

Step 08

Audit the five accuracies

Factual, contextual, visual, action, trust.

A right answer in the wrong form still fails. Audit your component across five layers of accuracy:

Factual — are the numbers right? Contextual — right for this user right now? Visual — does hierarchy match real importance? Action — is the recommended next step appropriate? Trust — does the UI honestly communicate uncertainty?

If any of the five fails, the component is wrong even if the data is right.

Step 09

Design refinement & feedback

The user's exit when the answer's wrong.

Every gen-UI component needs an honourable exit. Two flavours: positive feedback (this was right — train the model) and refinement (this was close — let me adjust).

For Sales Insight: a quiet thumbs pair in the corner; on thumbs-down expand a small form with three categories. A second affordance: “Refine this” — turns the widget back into a prompt prefilled with its current state.

Refine over restart. Always.

If you can answer all nine of these for your own component, you've designed a real generative one — not a chat prompt with extra steps.

06Principles

Ten things to keep nearby.

A list of guardrails. Each is the inverse of one common way generative UI fails. Print it, pin it, return to it.

01

Constrain the model

Allowlists over free text. Schemas over JSON. Tokens over hex codes.

02

Design the empty state

If you haven't drawn the “no answer yet” screen, you haven't started.

03

Surface uncertainty

If the model is guessing, the UI should say so — quietly, but clearly.

04

Density follows weight

Simple answers → Micro-UI. Complex answers → Macro-UI. Same grammar, different scale.

05

Composed, not spat out

Streaming should look choreographed. Stagger arrivals. Earn attention.

06

Familiarity wins

Repeat the same shape across sessions. Muscle memory beats novelty.

07

One choice per moment

The model decided what matters. Give the user one action that follows.

08

Legible thinking

Show progress, sources, and what the model is doing. Not just a spinner.

09

Refine, don't restart

The user shouldn't have to retype. Wrong answers turn into prefilled prompts.

10

Trust comes from receipts

Source. Freshness. Confidence. Three small lines. Non-negotiable.

07Failure modes

The ways it breaks.

Six common failure modes that ship in production more often than they should. Each is the inverse of a principle — worth recognising on sight.

Pitfall 01

Visual hallucination

A layout that looks beautiful but misrepresents the data — wrong chart for the question, hierarchy out of proportion to importance, decorative colour mistaken for risk signal. The fix is constrained primitives and visual guardrails.

Pitfall 02

Numbers without sources

A bold figure with no citation is worse than no answer. Make source links structurally mandatory at the schema layer — the component cannot render without them.

Pitfall 03

Generic spinner for everything

The user can't tell if their request was understood. Replace the spinner with a skeleton that mirrors the eventual layout — they know what's coming before it arrives.

Pitfall 04

Prose where structure works

If the answer is “a number, a trend, a source”, render a metric tile. Don't dump a paragraph. Structure is the gift gen-UI gives you over plain chat.

Pitfall 05

Every screen looks different

Personalisation taken too far destroys muscle memory. The fix is a stable spine — chrome, slots, choreography — with only the content variable.

Pitfall 06

No way to refine — only retry

The user has to retype their prompt every time. The fix is “refine this” — the current state becomes the prefilled basis for the next attempt.

08Iteration

Evaluation and iteration.

A generative component is never finished. It improves the way a design system improves — slowly, deliberately, through observation and iteration of the vocabulary.

01 · Set

Build a golden set

30–100 prompts you must always get right. Hand-curated. Versioned.

02 · Run

Watch real users

Lab preference scores lie. Watch completion rate, refinement count, abandonment.

03 · Read

Read the feedback

Categorise every thumbs-down. Look for clusters — they point at a missing primitive.

04 · Iterate

Update the vocabulary

A new slot, a tighter enum, a better empty state. Not prompt engineering.

You ship a vocabulary. You iterate the vocabulary. The model just speaks it.
